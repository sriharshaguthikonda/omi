# P5.1 — Moonshine transcript revision-window knob

> **For agentic workers:** Codex CLI implements, Claude orchestrates and corrects. One logical feature commit (not per-file). Steps use checkbox (`- [ ]`).

**Goal (Sri, verbatim):** *"the advantage of moonshine is that when it hears a longer sentence it goes back and changes the previous/upcoming tokens… we should be able to adjust the duration for which each transcript chunk is present or saved."* Expose Moonshine's **revision window** — how long a transcript line stays revisable (partial) before it locks (final) — as a user-tunable in **Settings → Transcription**. Longer window = more back-correction/context; shorter = snappier locking.

## Why this is cheap + isolated

- The Moonshine SDK line lifecycle is already wired: native `MoonshineListener.onLineTextChanged` → `isFinal=false` (partial, revisable), `onLineCompleted` → `isFinal=true` (locked). Dart bumps `_lineIndex` on final so the next line appends ([on_device_moonshine_socket.dart:159-175](../app/lib/services/sockets/on_device_moonshine_socket.dart)). **The revision/commit timing is decided by the SDK, not our code** — it's driven by `TranscriberOption(key,value)` string pairs and/or the model's `streaming_config.json`.
- Current options set: only `identify_speakers=false`, `return_audio_data=false` ([MoonshineSttPlugin.kt:75-79](../app/android/app/src/main/kotlin/com/friend/ios/MoonshineSttPlugin.kt)). Adding the window is **one more `TranscriberOption`** (or one `streaming_config.json` field), plumbed from a pref.
- Touches Transcription-settings + Moonshine socket/plugin only — **no overlap** with the in-flight `feature/bt-trigger-matrix` branch (BT triggers / dev-settings / registry). Merges independently.

## Architecture

```
Settings → Transcription (Moonshine section)
  slider "Transcript revision window"  →  SharedPreferencesUtil.moonshineRevisionWindowMs (int, ms; 0 = SDK default)
     → transcription_service.dart builds OnDeviceMoonshineSocket(revisionWindowMs: …)
        → socket.connect() passes it in the `initialize` MethodChannel args
           → MoonshineSttPlugin.initialize reads it → sets the matching TranscriberOption / streaming_config field
```

**Default = 0 = do not override the SDK.** Behaviour is unchanged unless the user opts in (protects the accumulation fix Sri just confirmed works, commit `65f176c`).

## Global Constraints

- Package `com.friend.ios`; Kotlin dir `com/friend/ios/`. No new MethodChannel — extend the existing `com.omi/moonshine_stt` `initialize` args.
- All user-facing strings via l10n (`context.l10n.*`), ARB keys added with `jq` (never read full ARB). Translate all locales (`ls app/lib/l10n/app_*.arb` minus `app_en.arb`); `flutter gen-l10n` must emit **0** untranslated warnings.
- Settings UI honours the repo UI rules: **never purple**; the setting must render correctly in System / Dark / Light theme.
- Formatting: `dart format --line-length 120`; Kotlin via pre-commit hook.
- **One feature commit.** Codex does NOT commit and does NOT edit ROADMAP/plans/Q&A/HANDOFF — orchestrator (Claude) reviews then commits the whole layer as one commit.
