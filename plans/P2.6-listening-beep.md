# P2.6 — Feedback beeps + "still listening" pulse (restored)

> **For agentic workers:** Standing rule — Codex CLI implements, Claude orchestrates and corrects. Steps use checkbox (`- [ ]`) syntax. This restores a feature Sri asked for that the P2 plan dropped.

**Goal (Sri, verbatim intent):** audible confirmation that capture is happening — *"beep during listening"*. Concretely:
1. Single soft beep on **start**, double soft beep on **stop**.
2. Optional **periodic "still listening"** low-volume beep while recording (so you know it's still on).
3. **Calm / not loud** — low base volume, ambient-noise-adjusted so it stays audible but never harsh.
4. All knobs live in **Developer settings** (enable, interval, base volume, ambient-adaptive, haptic).
5. Optional haptic pulse paired with each beep.

**Why native (not a Flutter audio package):** the still-listening pulse must keep firing while the app is backgrounded and the screen is off — that only survives in the capture foreground-service process. Use Android `ToneGenerator` (stdlib, zero new dependency). `ponytail:` no audio-player dep, ToneGenerator covers short tones natively.

## Architecture (reuse the shipped seam)

- Native `TriggerFeedback` object (Kotlin) owns a `ToneGenerator(STREAM_NOTIFICATION, volume)` + `Vibrator`.
- Start/stop beeps: Dart `TriggerRouter` (`app/lib/services/capture/trigger_router.dart`) already knows resolved `start`/`stop` — it invokes native feedback over a MethodChannel (or folds into the existing `com.friend.ios/trigger_actions` channel with a new method `feedback`).
- Still-listening pulse: a coroutine timer inside the capture foreground service (`OmiBleForegroundService.kt` / the phone-mic capture path), started when recording begins, cancelled on stop. Fires a short low tone every `intervalSec`.
- Ambient volume: the mic PCM already flows through `app/lib/services/audio_sources/phone_mic_source.dart` (16 kHz PCM16, 320-byte frames). Compute a rolling RMS, map to a **calm volume band** (e.g. floor..ceil of ToneGenerator's 0–100), pass to `TriggerFeedback`. `ponytail:` simple RMS→band map, not a full loudness model — upgrade only if it feels wrong on-device.
- Settings persisted in `SharedPreferencesUtil` (`app/lib/backend/preferences.dart`); UI in the Developer settings page.

## Settings (Developer settings, all persisted)

| Key | Default | Meaning |
|---|---|---|
| `feedbackBeepsEnabled` | true | master start/stop beep toggle |
| `stillListeningBeepEnabled` | false | periodic pulse while recording |
| `stillListeningIntervalSec` | 30 | pulse period |
| `feedbackBaseVolume` | low (e.g. 25/100) | base ToneGenerator volume |
| `feedbackAmbientAdaptive` | true | scale volume by ambient RMS |
| `feedbackHapticEnabled` | false | vibrate with each beep |
